# 서명 생성 모델: 초기 구현과 개선 과정

## 개요

Graves(2013)의 "Generating Sequences With Recurrent Neural Networks" 논문을 기반으로 서명/손글씨 생성 모델을 구현하고, 발견된 문제점을 분석하여 개선한 과정을 기록한다.

---

## 1. 초기 구현: Graves 아키텍처

### 핵심 구조

```
Input (x, y, pen) + Attention Window
        ↓
    LSTM Layer 1 (400 units, Peephole)
        ↓
    Soft Window Attention
        ↓
    LSTM Layer 2 (skip connection from L1)
        ↓
    LSTM Layer 3 (skip connection from L2)
        ↓
    MDN Output (20 Gaussian mixtures)
```

### 구현 파일 구조

| 파일 | 역할 |
|------|------|
| `models/synthesis.py` | 전체 네트워크 조합 |
| `models/lstm.py` | Peephole LSTM |
| `models/attention.py` | Location-based Soft Window |
| `models/mdn.py` | Mixture Density Network |
| `training/losses.py` | NLL Loss |
| `inference/sampler.py` | 추론 및 SVG 변환 |

### MDN 출력 파라미터

- `π` (pi): mixture weights (softmax)
- `μ` (mu): x, y 좌표 평균
- `σ` (sigma): x, y 표준편차 (exp → 양수)
- `ρ` (rho): x-y 상관계수 (tanh → [-1, 1])
- `e` (eos): end-of-stroke 확률 (sigmoid)

## 2. 문제 발견: 대각선 Drift

### 현상

- **낮은 Sharpness(bias)**: 문자 형태가 어느 정도 보임
- **높은 Sharpness(bias)**: 꼬불꼬불한 대각선으로 폭주

### 원인 분석

Graves 논문의 경고:
> "predictions can easily drift away from the manifold where training data lies"

**Autoregressive 모델의 구조적 한계:**
1. 한 점 예측 → 다음 점 입력 → 에러 누적
2. 초기 학습에서 분포가 불안정할 때 bias를 높이면 극단값 선택
3. 한 번 궤도 이탈하면 복구 불가 → 대각선 직선화

### Sampling 코드 분석

```python
# 문제가 된 부분
temp = max(temperature * (1.0 - bias), 0.1)
sigma_adjusted = sigma * temp  # bias=1이면 sigma가 0.1배로 축소
```

`bias=1`일 때 `temp=0.1` → sigma가 과도하게 줄어들어 거의 deterministic한 sampling이 됨. 모델이 아직 학습 중이라 "평균"이 의미없는 방향(대각선)을 가리킴.

## 3. 관련 연구 조사

### 기초 논문

**Alex Graves, "Generating Sequences With Recurrent Neural Networks" (2013)**
- LSTM + MDN 조합의 원조
- Location-based attention으로 텍스트 조건부 생성

### 알려진 실패 패턴

| 문제 | 원인 | 해결 방향 |
|------|------|----------|
| Long sequence drift | Autoregressive 에러 누적 | Diffusion 기반 비순차 생성 |
| Mode collapse | GAN 기반 접근 | VAE/Diffusion |
| Pen-up 분절 실패 | 이진 분류 불안정 | 별도 loss 가중치 |

### 최신 연구 동향 (2024-2025)

| 논문 | 핵심 아이디어 |
|------|--------------|
| **DiffInk** (2025) | Latent Diffusion Transformer, InkVAE로 glyph/style 분리 |
| **DiffusionPen** (ECCV 2024) | 5-shot 스타일 학습, Metric Learning |
| **DiffBrush** (ICCV 2025) | Full text-line 생성, Content-decoupled style |
| **DeepWriteSYN** (AAAI 2021) | Seq2Seq VAE, stroke 단위 합성 |

**공통점**: Autoregressive → Non-autoregressive/Diffusion 전환으로 drift 문제 구조적 회피

## 4. 개선 적용

구조 변경 없이 현재 아키텍처에서 할 수 있는 것들 우선 적용.

### 4.1 Sigma 최소값 보장

**변경 전:**
```python
sigma_adjusted = sigma * temp
```

**변경 후:**
```python
sigma_adjusted = sigma * max(temp, 0.5)
```

**효과:** 높은 bias에서도 최소 50% variance 유지 → 완전 deterministic 방지

### 4.2 EOS Loss 가중치 증가

**변경 전:**
```python
loss = stroke_nll + eos_bce  # 1:1
```

**변경 후:**
```python
loss = stroke_nll + 2.0 * eos_bce  # 1:2
```

**효과:** Pen-up 시점 학습 강화 → "끊어야 할 때 끊기"

### 4.3 EOS Sampling 선명화

**변경 전:**
```python
eos_sample = (torch.rand(...) < eos).float()
```

**변경 후:**
```python
eos_threshold = eos ** (1.0 / max(temp, 0.5))
eos_sample = (torch.rand(...) < eos_threshold).float()
```

**효과:**
- `bias=0`: `eos^1.0` = 원래대로
- `bias=1`: `eos^2.0` = 애매한 신호는 억제, 확실한 신호는 유지
